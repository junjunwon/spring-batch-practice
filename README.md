# 🔥 Spring Batch Practice Project 🔥

## 🎯 프로젝트 목적
이 프로젝트는 **Spring Batch를 주제별 Job 단위로 학습·실습하기 위한 스터디 프로젝트**입니다.

하나의 완성된 시스템을 만드는 것이 아니라,

- 특정 주제를 정해 Job을 만들고
- 필요하다면 기존 Job을 개선하거나 확장하고
- 학습 과정에서 나온 고민과 실험 코드를 그대로 남기는

방식으로 **Job 중심의 학습 기록을 축적**하는 것이 목적입니다.

---

## 🧪 스터디 방식

### 1. Job 중심 학습 🧩
이 프로젝트에서 **학습의 최소 단위는 Job**입니다.

- 하나의 Job = 하나의 학습 주제
- Job마다 필요한 구성요소만 포함
- Job 단위로 자유롭게 실험 / 개선 가능

> 새로운 주제를 학습할 때마다  
> `job` 하위에 새로운 패키지를 생성합니다.

---

### 2. 점진적 확장과 개선 🔁
스터디는 다음 두 가지 방식으로 진행됩니다.

- **새로운 Job 추가**
    - 새로운 개념 / 흐름 / 기능을 실험
- **기존 Job 개선**
    - 구조 리팩토링
    - 구현 방식 변경
    - 비교 실험 코드 추가

기존 코드를 “버리지 않고 남기는 것”을 원칙으로 합니다.

---

### 3. Job README 운영 방식 📝

각 Job 패키지에는 `README.md`를 포함합니다.

README에는 다음 내용을 자유롭게 작성합니다.
- 이 Job을 만든 이유
- 실습하면서 확인한 동작
- 고민했던 포인트
- 이후 개선하고 싶은 방향

정답이나 결론이 없어도 괜찮으며,  
학습 과정 자체를 남기는 것을 목표로 합니다.

Job의 동작을 전부 설명하지 않아도 되며,  
**“왜 이런 구조로 작성했는지”** 를 중심으로 기록합니다.

---

## 🗂️ 패키지 구조

```text
com.dev.batchpractice
├── common : 여러 Job에서 공통으로 사용하는 구성 요소를 모아두는 영역
│   ├── config
│   ├── listener
│   ├── util
│   └── support
│
├── domain : Batch 처리의 대상이 되는 도메인 영역
│   ├── entity
│   ├── repository
│   └── service
│
├── jobs : 학습 및 실습 단위의 Job을 주제별로 분리하여 관리
│   ├── simple-tasklet
│   │   ├── config
│   │   ├── step
│   │   ├── tasklet
│   │   └── README.md
│   │
│   ├── job-parameter-flow
│   │   ├── config
│   │   ├── step
│   │   ├── tasklet
│   │   ├── validator
│   │   └── README.md
│   │
│   └── ...
│
└── schedule : 외부 트리거 기반으로 Batch Job을 실행하기 위한 영역
    ├── config
    └── launcher
```
